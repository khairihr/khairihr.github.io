Selected work

Kuota Pesan — usage-based billing engine

in production

Some partners pay Meta's messaging fees through the company's own card, so any send that wasn't metered was money quietly lost. I built the engine that prices each message from Meta's pricing data and deducts prepaid credit in real time from the billing webhook — idempotent and deadlock-safe, so it holds up against live traffic. It rolled out in stages (report, then charge, then block) behind safety gates, with reconciliation jobs to recover historical leakage.

Laravel · MySQL · Meta Graph API · scheduled & queued jobs

AI sales-funnel labeling

paid feature

A feature that reads each contact's conversation and labels their funnel stage using an LLM. A dedicated worker runs it rate-limited and distinguishes "out of credit" from "rate-limited" so it never wastes spend retrying a call that can't succeed. Stage labels show up inline in the chat list; usage is metered and billed.

OpenAI API · queue workers · Laravel

Meta Flow webhook bridge

in production

Reworked how Meta Flow callbacks are handled so each receiver URL carries its own secret — the way Stripe signs webhooks — letting a single tenant route different flows to different backends. I removed the N+1 queries on the hot path, added caching and timeouts, and built a self-serve fix for when a tenant's public key drifts out of sync (previously a support ticket every time).

Webhooks · HMAC signing · caching

Partner API platform

used by partner orgs

The external REST API that zakat and wakaf organizations use to push transactions and contacts and pull back reports. I added API-key auth with IP allow-listing, idempotency to stop double-posting, outbound webhooks, and a provisioning endpoint that stands up a whole new tenant — company, user, and credentials — on its own. Documented publicly so partners can self-serve.

REST · API keys · multi-tenant · GitBook docs

Performance on a million-row dataset

ongoing

The contacts table runs into the millions of rows, so a lot of the work is keeping things fast: rewriting queries so they don't load everything into memory, adding the right indexes, removing N+1s, caching the dashboard, and moving heavy imports and media processing into background jobs.

MySQL · Laravel queues · caching

Also built: a support-ticketing module (reply templates, internal notes, live updates); partner self-onboarding with subscriptions and billing cycles; a Datadog observability setup with SLAs and structured logging; and the design-system refresh across the app.
