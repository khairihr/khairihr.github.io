Selected work
Billing

Kuota Pesan, a usage-based billing engine

Some partners pay Meta's messaging fees through the company's own card, so any send that wasn't metered was money quietly lost. I built the engine that prices each message from Meta's pricing data and deducts prepaid credit in real time from the billing webhook: idempotent and deadlock-safe, so it holds up against live traffic. It rolled out in stages (report, then charge, then block) behind safety gates, with reconciliation jobs to recover historical leakage.

AI feature

AI sales-funnel labeling

Reads each contact's conversation and labels their funnel stage using an LLM. A dedicated worker runs it rate-limited and tells the difference between "out of credit" and "rate-limited", so it never wastes spend retrying a call that can't succeed. Labels show up inline in the chat list; usage is metered and billed.

Integration

Meta Flow webhook bridge

Reworked how Meta Flow callbacks are handled so each receiver URL carries its own secret, the way Stripe signs webhooks, letting one tenant route different flows to different backends. I removed the N+1 queries on the hot path, added caching and timeouts, and built a self-serve fix for when a tenant's public key drifts out of sync (previously a support ticket every time).
